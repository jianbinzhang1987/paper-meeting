# 无纸化会议系统管理端未完成事项

## 说明

- 范围以 `paper-meeting-vue3` 管理端和 `paper-meeting-server/yudao-module-meeting` 会议后台为主。
- 判定方式以“需求文档/设计文档已有要求”对照“当前代码已落地能力”。
- 本文件只保留仍未完成或仅完成基础版的事项；本轮已补上的菜单顺序和表决控制接入不再列为未完成。

## P0：影响上线闭环，优先继续完成

- [ ] 独立“表决控制中心”页面未落地。
  当前只有会议详情弹窗内的表决列表与基础控制，未形成管理端独立的会中控制入口，也缺少对特定议题投票的集中监控视角。
- [ ] 已归档会议导出能力不完整。
  现在仅能导出会议资料和表决结果，需求中提到的签到表、手写批注资料备份尚未看到管理端落地。
- [ ] 公共资料库仍是平面列表。
  现有页面支持分类字段录入，但还没有“公共目录树、上传、预览、长期资料管理”的完整体验，仍偏向数据维护页。
- [ ] 安装包管理未形成真实“上传-启用-端侧检查更新”闭环。
  当前是下载地址录入模式，没有安装包上传、版本文件校验联动、检查更新指令下发能力。
- [ ] 客户端样式管理缺少预览与推送闭环。
  现在可维护颜色、字号、背景、Logo 并启用，但没有可视化预览、样式校验、向终端推送生效反馈。

## P1：需求已写入，但管理端仍未真正落地

- [ ] 网站贴牌与菜单编辑未落地为会议系统专属能力。
  需求文档里提到“网站名称、LOGO、侧边栏菜单文字”可配置，目前仅有通用系统菜单管理，没有会议系统级的贴牌配置页。
- [ ] 用户组管理能力未对齐会务场景。
  需求提到跨部门用户组，用于建会批量邀请和授权；当前未看到会议域下专用用户组页面或快捷选组能力。
- [ ] 会议资料库缺少访问审计和资料使用追踪。
  设计文档已把公共资料定位为长期共享资源，目前未看到下载、浏览、启停使用痕迹的追踪能力。
- [ ] 会中消息缺少发布追踪视角。
  已有消息草稿/发布，但没有已读、送达、失败、终端回执等状态统计。

## P2：可用性与权限治理仍需补齐

- [ ] 菜单分组仍是单层结构。
  本轮已重排顺序，但若后续模块继续增加，建议拆成“会议工作台 / 会中服务 / 资源配置 / 终端运维”四组，否则左侧菜单还会再次变乱。
- [ ] 会议系统权限点仍需继续细化复查。
  本轮补了 `agenda / attendee / file / vote` 权限节点，但仍建议把导出、发布、强制同屏等高风险动作拆出独立权限。
- [ ] 会议首页仍偏总览，不含快捷告警。
  现在能看会议室占用和今日会议，但还没有会议冲突、高优先级审批、即将开始会议的提醒卡片。
- [ ] 管理端缺少系统化验收清单。
  目前没有看到针对“预约 -> 审批 -> 会前配置 -> 会中控制 -> 归档导出”的端到端验收文档或测试用例。

  调整为“会议首页 -> 我的会议 -> 预定审批 -> 会议列表 -> 已归档会议 -> 会议模板 -> 会议室管理 -> 会中消息 -> 公共资料库 -> 客户端样式 -> 安装包管理”。

  复用现有后端接口，先把已具备服务端能力但前端未接上的表决控制补齐。
