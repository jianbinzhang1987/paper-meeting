# 会议实时协议

## 通道

- WebSocket 路径：`/infra/ws`
- 鉴权方式：`?token=<accessToken>`
- 消息结构：

```json
{
  "type": "meeting-sync-request-send",
  "content": "{\"meetingId\":1}"
}
```

## 服务端下发消息

### 通知

- `meeting-notice-published`
- 载荷字段：
  - `meetingId`
  - `noticeId`
  - `title`
  - `content`
  - `publishedTime`

### 表决

- `meeting-vote-started`
- `meeting-vote-finished`
- `meeting-vote-published`
- 载荷字段：
  - `meetingId`
  - `voteId`
  - `title`
  - `status`

### 全局控制

- `meeting-force-return`
- `meeting-force-logout`
- 载荷字段：
  - `meetingId`
  - `operatorUserId`
  - `operatorName`
  - `targetUserId`
  - `targetUserName`
  - `content`

### 同屏

- `meeting-sync-request`
- `meeting-sync-approved`
- `meeting-sync-rejected`
- `meeting-sync-stopped`
- 载荷字段：
  - `meetingId`
  - `requestId`
  - `requesterUserId`
  - `requesterName`
  - `requesterSeatName`
  - `documentId`
  - `documentTitle`
  - `page`
  - `status`
  - `approverUserId`
  - `approverName`

说明：
- `status` 统一使用 `pending / approved / rejected / stopped`
- `requestId=all` 表示全局强制停止

### 服务呼叫

- `meeting-service-requested`
- `meeting-service-updated`
- 载荷字段：
  - `meetingId`
  - `requestId`
  - `requesterUserId`
  - `requesterName`
  - `requesterSeatName`
  - `category`
  - `detail`
  - `status`
  - `handlerUserId`
  - `handlerName`

说明：
- 当前状态值已使用 `pending / processing / completed`
