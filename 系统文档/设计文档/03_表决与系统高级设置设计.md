# 详细设计说明书 - Phase 3: 表决与系统高级设置

## 1. 表决系统设计

### 1.1 表决流程控制
1.  **创建/引用**：从 `mt_vote_template` 快速生成表决议题。支持单选、多选、实名、匿名。
2.  **会中控制 (秘书/后台)**：
    -   **开始**：下发 `START_VOTE` 指令，客户端弹出遮罩层显示表决项。
    -   **提醒**：向 `sign_status=1` 且 `vote_status=0` 的用户发送震动提醒。
    -   **结束**：停止计票。
3.  **结果统计**：
    -   **实时率**：`(参与人数 / 应到人数) * 100%`。
    -   **通过率**：`(赞成票 / 总投有效票)`。
    -   **可视化**：生成饼图/柱状图数据 (ECharts JSON)。

## 2. 系统全局配置

### 2.1 客户端 UI 动态换肤
1.  **参数化配置**：后台提供取色器设置 `primary-color`、`bg-image-url`、`font-size-base`。
2.  **同步机制**：配置保存后，通过 WebSocket 或客户端心跳拉取全量 CSS 变量 JSON。
3.  **多主题预设**：中性风、科技蓝、政务红。

### 2.2 安装包 (APK) 自动化分发
1.  **版本矩阵**：维护 `mt_apk_version` 表 (versionCode, MD5, forced_update, apk_url)。
2.  **增量提示**：客户端请求 `checkUpdate`，若版本滞后则弹窗提示下载。

## 4. 会议模板与克隆
- **模板沉淀**：会议归档前支持“另存为模板”。
- **深层克隆**：复制议程结构、表决配置、人员名单（不复制文件物理副本，仅引用）。

## 5. 开发指引
- **后端代码**：存放于 `ruoyi-modules-meeting` (表决) 与 `ruoyi-admin` (设置) 模块。
- **前端视图**：存放于 `src/views/meeting/control` (会中控制) 与 `src/views/system/config`。
