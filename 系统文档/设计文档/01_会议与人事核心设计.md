# 详细设计说明书 - Phase 1: 会议与人事核心

## 1. 业务流程设计

### 1.1 会议预约与创建流程
1.  **冲突检查**：管理员选择会议室与时间段，系统调取 `mt_meeting` 表检查该会议室在目标时段内是否存在“进行中”或“已预约”的冲突记录。
2.  **基本信息录入**：支持手动录入或从 `mt_template` 复制。包括会议名称、地点、等级（正常/保密）、控制类型（秘书/无）。
3.  **议程编排**：支持多级树状议程。主议题下可挂载子议题，每个议题关联 `agenda_id`。
4.  **参会人员绑定**：从 `sys_user` 或 `mt_user_group` 批量导入。存储于 `mt_participant`，并分配初始角色（主持人、秘书、与会者）。

### 1.2 图形化选座流程
1.  **模板载入**：根据 `room_id` 获取 `mt_room` 中的 `layout_json`（包含座位坐标与设备ID）。
2.  **人员拖拽**：前端通过拖放组件将参会人名单与座位节点匹配。
3.  **状态保存**：将 `user_id` 更新至 `mt_participant` 的对应 `seat_no` 字段。

## 3. 功能模块详细说明

### 3.1 会议室管理
- **座位设计师**：支持在后台定义会议室的物理布局。采用网格化或自定义坐标系。
- **设备绑定**：每个座位需绑定一个固定设备ID（或IP/MAC），用于免密登录时的座位自动识别。

### 3.2 人事增强 (RuoYi 扩展)
- **用户组功能**：新增 `mt_user_group` 与 `mt_user_group_rel` 表。
- **模板导入**：支持 Excel 批量导入。
