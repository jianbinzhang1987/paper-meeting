# 无纸化会议系统平板端 UI/UX 改版设计方案

## 1. 设计目标

本方案面向无纸化会议系统平板端，重点解决当前界面存在的以下问题：

- 视觉层级混乱，重点内容不突出
- 组件风格偏表单系统，缺乏现代感
- 按钮、卡片、边框、导航缺乏统一设计语言
- 不符合横屏平板操作习惯，点击热区偏小
- 空间利用效率低，页面留白和层次关系失衡
- 缺乏会议场景氛围，不够沉稳、专注和高级

本次改版目标：

- 建立适合政企会议场景的平板端设计语言
- 强化文稿阅读页的核心地位
- 优化会议信息页和终端初始化页的视觉与交互体验
- 为秘书控制台等高频操作页面建立更清晰的控制中心结构
- 提供可直接指导 Flutter 落地的布局和组件规范

## 2. 整体视觉风格规范

### 2.1 风格定位

整体风格建议定义为：

`深色会议外壳 + 浅色内容台面 + 低饱和商务蓝灰体系`

设计关键词：

- 克制
- 稳重
- 清晰
- 专注
- 专业

视觉参考方向：

- iPad Pro 企业应用
- 飞书高阶工作台
- Notion 的清晰层级
- 钉钉高端政企工作台

### 2.2 色彩建议

- 页面背景：`#EEF3F8`
- 顶部框架深色层：`#16324F`
- 左导航深蓝灰：`#1D3147`
- 主强调蓝：`#2F6BFF`
- 辅助蓝灰：`#6F8296`
- 卡片底色：`#FFFFFF`
- 轻分割线：`#DCE4EC`
- 成功色：`#2D8C5B`
- 警示色：`#D58A1F`
- 危险色：`#C94B4B`

使用原则：

- 高亮蓝只用于一个页面内最关键的主操作
- 大面积区域避免纯白和高对比蓝
- 会议主框架尽量采用深蓝灰，以增强场景感

### 2.3 字体层级

- 页面主标题：`32 / SemiBold`
- 区块标题：`22 / Medium`
- 卡片标题：`18 / Medium`
- 正文：`15 / Regular`
- 辅助信息：`13 / Regular`
- 标签文字：`12 / Medium`

### 2.4 圆角与阴影

- 标准控件圆角：`12`
- 卡片圆角：`16`
- 浮层圆角：`20`
- 标签圆角：`999`

阴影统一为轻量单层：

- `0 10 30 rgba(22, 50, 79, 0.08)`

### 2.5 间距系统

- 页面外边距：`24`
- 模块间距：`20`
- 卡片内边距：`20`
- 组件间距：`12`
- 控件高度：`44 / 48`

### 2.6 平板交互基线

- 所有点击热区不低于 `44x44`
- 横屏为主，所有主页面优先采用分栏布局
- 常用操作不超过两次点击
- 主信息区始终优先于工具区

## 3. 顶层框架改版

当前系统顶层框架的问题是左导航、顶部状态和内容标题处于同一视觉层级，导致主次关系不清。

### 3.1 建议结构

```text
┌────────────────────────────────────────────────────────────┐
│ 顶部状态带 72px                                            │
├──────────────┬─────────────────────────────────────────────┤
│ 左导航 92px   │ 页面内容区                                  │
│ 图标优先      │ 标题区 + 主内容                             │
│ 可展开文字     │                                             │
└──────────────┴─────────────────────────────────────────────┘
```

### 3.2 左侧导航

- 默认宽度：`92`
- 展开宽度：`220`
- 导航项高度：`56`
- 默认仅显示图标，选中项显示浅蓝胶囊底
- 点击顶部标识或长按时展开显示文字
- 导航背景采用深蓝灰或浅灰台面，不建议继续大面积白底

选中态建议：

- 背景：`#DCEBFF`
- 图标颜色：`#16324F`
- 文字颜色：`#16324F`

### 3.3 顶部状态栏

- 高度：`72`
- 左侧保留会议名作为系统标识
- 中间放状态胶囊组：网络、实时、同步时间
- 右侧放当前账号、主题切换、退出会议

状态胶囊统一：

- 高度：`40`
- 圆角：`12`
- 边框：`1px #DCE4EC`
- 深色框架下可使用浅底胶囊提升识别度

### 3.4 页面标题区

- 每个页面内容区内部保留单独页面标题
- 高度建议：`88`
- 标题下增加 1 行说明文字
- 页面标题不再承担系统标题功能

## 4. 会议信息页改版方案

### 4.1 当前问题

- 议题区域只是普通列表，缺乏重点和状态表达
- 参会人员只是名单展示，角色和状态不够突出
- 内容区域过于空旷，没有形成有效信息焦点

### 4.2 页面布局

建议采用 `64% / 36%` 双栏结构：

```text
┌────────────────────────────────────────────────────────────┐
│ 页面标题：会议信息                                          │
│ 页面说明：查看会议简介、议题安排及参会人员名单              │
├───────────────────────────────┬────────────────────────────┤
│ 左 64%                        │ 右 36%                      │
│ 会议概览英雄卡                 │ 参会人员卡                  │
│ 议题卡片列表                   │ 按角色分组                  │
│ 会议资料摘要                   │ 签到/在线状态               │
└───────────────────────────────┴────────────────────────────┘
```

### 4.3 左栏设计

#### 4.3.1 会议概览英雄卡

- 高度：`180`
- 背景：浅灰蓝渐变或浅蓝灰底
- 左上：会议标题
- 下方：会议简介 2 行
- 底部信息条：会议室、时间、当前议题、文稿数量
- 右上可加入抽象会议图形或徽标

#### 4.3.2 议题列表卡片化

每个议题卡建议：

- 高度：`96`
- 左侧为序号圆牌：`01 / 02 / 03`
- 中间为议题标题和简要说明
- 右侧为状态标签：`进行中 / 待开始 / 已结束`
- 底部可显示文稿数量和预计时长

当前选中议题状态：

- 背景：`#F2F7FF`
- 左侧：`4px` 高亮蓝竖条
- 标题加深

#### 4.3.3 资料摘要区

- 放在议题列表下方
- 展示当前议题关联文稿 2 至 3 条
- 每条包括文稿名称、页数、摘要
- 点击直接跳转至阅读页

### 4.4 右栏设计

#### 4.4.1 参会统计区

放在人员列表上方：

- 应到人数
- 已签到人数
- 在线人数

#### 4.4.2 参会人员分组展示

分组：

- 主持人
- 秘书
- 参会人员

每个人员项建议高度 `72`：

- 左：头像圆形
- 中：姓名、角色标签、座位号
- 右：签到状态点或在线状态

### 4.5 低保真草图

```text
┌──────────────────────────────────────────────────────────────┐
│ 会议概览带 70%                          状态摘要 30%         │
├───────────────────────────────┬──────────────────────────────┤
│ 议题时间轴卡片列表 64%         │ 参会人员分组卡 36%           │
│ [01 年度经营分析][进行中]      │ [主持人] 张明 主席位 已签到   │
│ [02 项目推进][待开始]          │ [秘书]   李娜 秘书席 在线     │
│ [03 预算审议][待开始]          │ [参会人] 王强 A01 已签到      │
│ 资料摘要卡                      │ [参会人] 陈洁 A02 未签到      │
```

## 5. 文稿阅读页改版方案

### 5.1 当前问题

- 工具按钮过多且平铺，文稿不是视觉中心
- 左栏目录层级较弱
- 右侧工具区严重表单化
- 阅读控制区不够专业，不像专门的 PDF 阅读器

### 5.2 页面布局

建议采用三栏阅读工作台：

- 左栏：`18%`
- 中栏：`60%`
- 右栏：`22%`

```text
┌────────────────────────────────────────────────────────────┐
│ 阅读头部 64px                                              │
├────────────┬──────────────────────────────┬───────────────┤
│ 左目录18%   │ 中间阅读60%                  │ 右工具22%      │
│ 议题tab      │ 文稿标题条                    │ 批注/书签/协同  │
│ 文稿列表      │ PDF主画布                     │ tab切换         │
│ 缩略图/目录   │ 底部悬浮控制条                │ 当前页相关内容  │
└────────────┴──────────────────────────────┴───────────────┘
```

### 5.3 左栏设计

#### 5.3.1 议题切换区

- 高度：`44`
- 使用横向 tabs
- 当前议题高亮

#### 5.3.2 文稿列表区

每个文稿项：

- 高度：`68`
- 展示文稿名、页数、最近阅读页
- 当前文稿使用高亮背景

#### 5.3.3 目录与缩略图切换

支持两种模式：

- 目录模式：章节标题列表
- 缩略图模式：页面缩略卡

### 5.4 中间阅读区设计

#### 5.4.1 阅读头部

左侧：

- 文稿名称

中间：

- 页码
- 总页数
- 阅读比例

右侧：

- 当前同屏状态
- 一个主按钮
- 一个更多操作按钮

主按钮根据状态切换：

- 未同屏：`申请同屏`
- 审批中：`等待审批`
- 已同屏：`结束同屏`
- 跟随中：`退出跟随`

#### 5.4.2 PDF 主画布

- 背景使用浅灰蓝阅读底
- 纸张居中显示
- 页面最大宽度：`680~760`
- 增大四周留白
- 水印透明度建议：`0.04~0.06`

#### 5.4.3 底部悬浮控制条

不要采用裸露 Slider 结构，建议改成胶囊浮条：

- 上一页
- 页码输入
- 总页数
- 进度条
- 下一页
- 适宽
- 适页
- 100%

### 5.5 右侧辅助工具区设计

不要同时展示注释输入框、书签输入框和空列表区。建议使用 tabs：

- 批注
- 书签
- 协同

#### 5.5.1 批注 Tab

顶部工具条：

- 画笔
- 荧光笔
- 文字
- 橡皮
- 撤销

内容区：

- 当前页批注列表
- 最近批注

底部快捷动作：

- 新增文字批注

#### 5.5.2 书签 Tab

- 顶部按钮：`收藏当前页`
- 列表显示书签名称、页码、时间
- 长按支持重命名

#### 5.5.3 协同 Tab

- 最近实时消息
- 同屏申请状态
- 跟随状态说明

### 5.6 交互路径优化

#### 5.6.1 批注

文字批注路径：

1. 点击 `批注`
2. 选择 `文字批注`
3. 在 PDF 页面点击位置
4. 弹出小输入浮层
5. 输入完成后直接生成批注并自动保存

手写批注路径：

1. 点击 `画笔`
2. 直接在页面书写
3. 自动保存
4. 双指负责平移与缩放

#### 5.6.2 同屏

推荐状态流：

1. 默认显示 `申请同屏`
2. 点击后显示 `等待审批`
3. 审批通过后顶部显示状态条 `正在同屏展示`
4. 跟随他人同屏时提示 `正在跟随主席视图`
5. 退出时弹出确认

### 5.7 手势建议

文稿页必须支持：

- 单指点击聚焦
- 左右轻扫翻页
- 双指缩放
- 双指平移
- 双击切换 `适宽 / 100%`

开启批注模式后：

- 单指改为书写
- 双指仍保留平移缩放

### 5.8 低保真草图

```text
┌──────────────────────────────────────────────────────────────┐
│ 文稿标题 / 页码 / 同屏状态 / 主操作                         │
├──────────────┬──────────────────────────────┬───────────────┤
│ 议题+目录18% │ PDF阅读区 60%                 │ 辅助工具22%    │
│ 议题Tab       │                               │ [批注][书签][协同]
│ 文稿列表       │         PDF Page              │ 当前页批注列表   │
│ 缩略图/目录    │         水印叠加层             │ 快捷添加批注     │
├──────────────┴──────────────────────────────┴───────────────┤
│ 上一页  页码输入  进度条  下一页  缩放  适宽/适页             │
└──────────────────────────────────────────────────────────────┘
```

## 6. 终端初始化页改版方案

### 6.1 当前问题

- 页面像一个普通登录卡片
- 左右内容失衡，整体显得空
- 缺少设备激活和会前准备的仪式感

### 6.2 布局建议

采用左右双区结构：

- 左区：`56%`
- 右区：`44%`

```text
┌────────────────────────────────────────────────────────────┐
│ 深蓝背景 / 设备激活页                                      │
│                                                            │
│ 左：会议空间氛围区           │ 右：签到/激活卡             │
│ 标题、设备图、会议室信息      │ 账号选择                    │
│ 当前设备状态时间线            │ 密码输入（如需要）          │
│ “会议已准备就绪”氛围文案      │ 主按钮：进入会议并签到      │
└────────────────────────────────────────────────────────────┘
```

### 6.3 左侧氛围区

- 顶部显示会议名
- 中部放设备插图或会议场景抽象图形
- 底部放设备信息卡：会议室、座位、服务器、终端编号
- 再加一条激活状态时间线：
  - 终端就绪
  - 连接会务服务
  - 等待签到
  - 进入会议

### 6.4 右侧签到卡

- 宽度：`420~460`
- 圆角：`20`
- 背景：白色
- 标题：`参会签到`
- 账号选择控件高度：`52`
- 如需密码，平滑展开密码输入框
- 主按钮固定在底部，全宽

### 6.5 Step Flow 版初始化页

如果改造的是设备激活流程而非签到流程，建议采用三步式：

1. 网络接入
2. 终端绑定
3. 完成激活

每一步只展示当前必须输入的字段，避免一次性展示所有配置项。

## 7. 秘书控制台改版方案

### 7.1 当前问题

- 虽然已经分卡片，但依旧是表单区块化拼接
- 控制台缺少“控制中心”的强弱主次
- 部分高风险操作和普通操作视觉等级接近

### 7.2 页面布局

建议采用 `55% / 45%` 双列工作台：

左列：

- 表决控制中心
- 同屏调度台
- 实时消息记录

右列：

- 呼叫服务后台处理
- 全局窗口强制控制中心

### 7.3 表决控制中心

建议展示：

- 当前表决标题
- 当前状态
- 已投人数
- 剩余人数

主操作仅保留：

- 开始表决
- 结束表决
- 发布结果

`重置表决` 放入更多操作，不与主操作同级平铺。

### 7.4 同屏调度台

每个申请卡显示：

- 申请人
- 文稿名称
- 申请时间
- 状态

右侧仅放两个操作：

- 同意
- 拒绝

### 7.5 实时消息记录

建议改成时间线结构：

- 图标
- 事件标题
- 时间
- 关联对象

### 7.6 呼叫服务后台处理

每个服务单做成 compact 卡片：

- 用户名
- 座位号
- 服务类型
- 请求详情
- 状态

状态切换建议使用 segmented control：

- 待处理
- 处理中
- 已完成

### 7.7 全局控制中心

广播消息区：

- 标题
- 内容
- 常用模板

操作分级：

- 主按钮：发送广播
- 次按钮：强制停止同屏
- 次按钮：强制返回同屏
- 危险按钮：强制注销签到

## 8. UI 组件设计规范

### 8.1 按钮

#### 主按钮

- 高度：`48`
- 圆角：`12`
- 背景：`#2F6BFF`
- 文字：白色
- 左右 padding：`20`

用途：

- 进入会议
- 申请同屏
- 完成激活
- 发送广播

#### 次按钮

- 高度：`48`
- 圆角：`12`
- 白底或浅灰底
- 边框：蓝灰边框

#### 弱按钮

- 高度：`40`
- 浅灰底
- 文字深色

#### 危险按钮

- 高度：`48`
- 圆角：`12`
- 浅红底
- 红色文字

#### 图标按钮

- 尺寸：`44 x 44`
- 用于阅读工具栏和状态栏

### 8.2 卡片

#### 信息卡

- 背景：白色
- 圆角：`16`
- 内边距：`20`
- 轻阴影

#### 列表卡

- 高度：`72~96`
- 圆角：`14`
- 默认边框：`1px #E4EAF0`

选中态：

- 背景：`#F2F7FF`
- 左侧高亮条

### 8.3 标签

#### 角色标签

- 主持人：深蓝底浅字
- 秘书：灰蓝底浅字
- 参会人员：浅灰底深字

#### 状态标签

- 成功：绿底绿字
- 待处理：黄底黄字
- 危险：红底红字

统一规范：

- 高度：`28`
- 左右 padding：`10`
- 圆角：`999`

### 8.4 输入框

- 高度：`48`
- 圆角：`12`
- 背景：`#F7F9FC`
- 默认无重边框
- 聚焦时显示 `1.5px` 蓝色描边

不建议继续使用强 Material 浮动 label，建议采用“字段标题 + 输入框”的垂直结构。

### 8.5 工具栏

- 高度：`56`
- 胶囊式浮层设计
- 图标 + 文本
- 每组最多 2 至 3 个操作

### 8.6 侧边栏

- 折叠态以图标为主
- 展开态显示文字
- 选中态用浅蓝块
- 不建议继续使用厚重的默认 NavigationRail 样式

## 9. Flutter 落地建议

### 9.1 是否需要单独引入 Flutter UI 组件库

结论：

`不建议为这个项目重度引入一整套第三方 UI 组件库作为主要设计基础。`

原因：

- 当前项目是政企平板会议系统，视觉风格需要高度定制
- 第三方组件库大多偏通用后台风或移动消费风
- 引入整套组件库会加快早期搭建，但容易把界面带回“表单系统”路线
- 后续想做统一的会议场景语言时，反而会被组件库默认样式限制

更合理的方案是：

- 基于 Flutter 原生 Material 体系做轻量二次封装
- 自建一套项目级设计 Token 和基础组件
- 仅在个别复杂控件场景引入第三方组件

### 9.2 推荐技术策略

#### 推荐方案：原生 Material 3 + 自定义主题 + 自建基础组件

建议建立：

- `MeetingThemeTokens`
- `MeetingColors`
- `MeetingSpacing`
- `MeetingRadius`
- `MeetingButton`
- `MeetingCard`
- `MeetingInput`
- `MeetingSidebarItem`
- `MeetingStatusChip`
- `MeetingSectionCard`

优点：

- 风格统一
- 可控性高
- 长期维护成本低
- 更适合政企项目演进

### 9.3 可以按需引入的第三方控件类型

适合引入第三方的场景：

- PDF 阅读器
- 图表

不建议依赖第三方 UI 库提供的：

- 按钮

## 10. 总结

本方案的核心思路不是简单“美化”，而是将系统从“通用后台界面”转为“会议平板工作台”。

改版后的目标状态应体现为：

- 左导航更克制
- 顶部状态更结构化
- 会议信息页更像会议概览中心
- 文稿页成为真正的核心工作区
- 初始化页具有设备激活的仪式感
- 秘书台更像控制中心而不是表单拼接页

在 Flutter 落地层面，推荐采用“原生 Material 3 + 自定义主题 Token + 自建基础组件”的方式，而不是直接套用一整套第三方 UI 库。
