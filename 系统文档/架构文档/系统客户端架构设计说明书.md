# 无纸化会议系统-客户端(Tablet)架构设计说明书

## 1. 总体设计原则
客户端作为与会者最直接的交互界面，架构设计核心关注：
*   **极致流畅：** 针对鸿蒙和安卓平板进行深度底层优化，确保文稿翻页和批注毫秒级响应。
*   **高可靠性：** 具备强大的离线查阅能力和断网自动重连恢复机制。
*   **低延时同屏：** 同屏同步延时需控制在 100ms 以内（局域网环境）。
*   **一板多能：** 需同时满足“普通参会”与“秘书主控”两种操作模式。

---

### 2.2 Flutter 核心技术选型库
基于 Flutter 框架，我们将集成以下经过商用验证的插件库：
*   **状态管理：** **Provider / Bloc** (针对大中型企业应用，确保业务逻辑与 UI 彻底解耦)
*   **文稿渲染引擎：** **pdfrx / flutter_pdfview**
    *   *说明：* 封装了高性能的 **Pdfium** 原生库，支持超大 PDF 的丝滑缩放、双指翻页及特定页面位置跳转。
*   **网络与通信：** **Dio** (HTTP 请求) + **web_socket_channel** (实时同屏信令)
*   **手写记录引擎：** **CustomPainter + Touch Events 序列化**
    *   *说明：* 通过 Flutter 自绘引擎实现 10ms 级笔迹渲染，通过串口或 Socket 实时传播笔迹坐标点（Point-to-Point Sync）。
*   **本地存储：** **Path Provider + Dio (File Download)** + **Hive / Sqflite** (持久化配置)
*   **安全加固：** **Flutter Securty Storage** (存储高保密 Token 与密钥)

---
*   **文档渲染引擎：** **Pdfium / MuPDF**
    *   *选择理由：* 高性能的原生 PDF 渲染库，支持海量页面的快速加载与缩放。
*   **手写批注引擎：** **基于 Native Canvas 的自定义绘图组件**
    *   *技术细节：* 采用双缓存绘制技术，支持压力感应、笔迹美化及批注轨迹序列化（供同步使用）。
*   **网络通信组件：** **OkHttp + WebSync (自定义 WebSocket)**
    *   *功能描述：* 离线下载使用 HTTP，实时指令同步使用高可靠的长连接。

### 2.3 数据持久化
*   **本地数据库：** **Room (Android)** / **Relational Store (HarmonyOS)**
*   **文件存储：** 具备 AES-256 加密的本地 Cache 目录，存储从服务器下发的离线文稿。

## 3. 核心专项架构设计：同屏同步机制

为了兼顾“性能”与“灵活性”，系统设计双重同步模式：

### 3.1 模式 A：指令级同步 (推荐模式)
*   **应用场景：** 文档浏览、翻页跟随、手写批注同步。
*   **实现原理：** 
    1.  主讲端（或主持人）仅广播“当前文件ID + 当前页码 + 缩放参数 + 批注轨迹数据”。
    2.  跟随端接收到指令后，在本地渲染引擎中自行执行跳转和绘制。
*   **优点：** 流量极小（KB级别），不占带宽，清晰度为本地无损。

### 3.2 模式 B：流媒体镜像同步 (兜底模式)
*   **应用场景：** 运行第三方 App 演示（如 WPS）、播放非系统内置格式视频。
*   **实现原理：** 
    1.  采用 **MediaProjection (Android)** 获取屏幕镜像。
    2.  通过 **H.264/H.265** 硬编码推送。
    3.  跟随端通过自建的简单多媒体接收器进行播控。
*   **优点：** 兼容性强，支持任意界面同步。

## 5. 关键业务流程

### 5.1 离线预加载流程
1.  **登录成功：** 客户端请求当前会议的资料清单。
2.  **差异化比对：** 检查本地 Cache 是否已存在对应版本的 MD5 校验文件。
3.  **静默下载：** 启动 Service 在后台根据优先级（当前议题优先）下载加密文件。
4.  **即点即开：** 用户点击议题时，直接从本地解密渲染，实现秒开。

### 5.2 断网重连恢复逻辑
1.  **心跳检测：** 客户端与后台每 5s 进行一次心跳。
2.  **重连策略：** 指数退避算法重连。
3.  **状态同步：** 重连成功后，主动向服务器索取“当前最新指令”，强制本地状态与主讲人同步（如自动翻至主讲人所在页）。

## 6. 性能目标要求
*   **冷启动时间：** < 2.0s
*   **大文档（500MB+）首屏加载：** < 1.0s
*   **翻页响应时延：** < 100ms
*   **系统内存占用：** 峰值控制在 500MB 以内（不含视频播放）。
